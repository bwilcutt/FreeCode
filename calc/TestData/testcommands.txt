# SIGNAL PROCESSING — FFT / IFFT
# Notes on verification strategy:
#   FFT outputs are stored as matrix variables ($fft_re etc.)
#   so scalar == comparisons are not directly applicable.
#   Where possible, Parseval's theorem is used via the
#   det(transpose(v)*v) trick, which turns the inner product
#   of a column vector into a printable scalar.
#
#   Parseval's theorem (DFT):
#     sum|x[n]|² = (1/N) * sum|X[k]|²
#   i.e.  det(Xᵀ·X) = N · det(xᵀ·x)
# ═══════════════════════════════════════════════════════════
# ── impulse [1 0 0 0 0 0 0 0] ───────────────────────────────

$imp_im
$imp_mag
$imp_phase

# sum of squared phases should be 0
# ── DC signal [2 2 2 2 2 2 2 2] ─────────────────────────────

$dc8_re
$dc8_im
